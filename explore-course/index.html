组装课程的关键脚本

真正把模块拼成完整页面的是一个脚本：`build.sh`。它把课程碎片拼成单个页面。

CODE 

# Assembles the course from parts.
# Run from the course directory: bash build.sh
set -e
cat _base.html modules/*.html _footer.html > index.html
echo "Built index.html — open it in your browser."
PLAIN ENGLISH

这段注释说明：脚本负责把课程的各个部分组装起来。

提示你在课程目录里运行它。

`set -e` 表示一旦有错误就停止，避免生成半成品。

`cat` 把壳、模块和页脚拼成最终的 `index.html`。

最后打印提示，告诉你可以打开课程了。

💡
关键洞察

生成流程是可重复的：你改完内容，再跑一次脚本就能得到新的课程。

技能说明书

SKILL.md 如何指挥整个课程生成

SKILL.md 就像一本剧本

在这个SKILL.md里，最重要的是课程的目录结构，它告诉你最终会生成哪些文件。

CODE 

course-name/
  styles.css       ← copied verbatim from references/styles.css
  main.js          ← copied verbatim from references/main.js
  _base.html       ← customized shell (title, accent color, nav dots)
  _footer.html     ← copied verbatim from references/_footer.html
  build.sh         ← copied verbatim from references/build.sh
  briefs/          ← module briefs (complex codebases only, can delete after build)
  modules/
    01-intro.html
    02-actors.html
    ...
  index.html       ← assembled by build.sh (do not write manually)
PLAIN ENGLISH

课程会在一个新文件夹里生成。

样式和交互脚本是直接复制来的，不需要重写。

`_base.html` 是课程壳，用来放标题和导航点。

模块内容放在 `modules/` 里，最后由 build.sh 拼成 index.html。

谁在和谁对话？

可以把整个流程想成一个小团队对话：你提出目标，SKILL.md 负责指挥，参考资料提供模板，模块文件写具体内容。

交互引擎负责细节体验

main.js 里有很多小引擎，其中一个负责把tooltip摆在合适的位置，不会被卡在容器里。

CODE 

  function positionTooltip(term, tip) {
    const rect     = term.getBoundingClientRect();
    const tipWidth = Math.min(320, Math.max(200, window.innerWidth * 0.8));
    let left = rect.left + rect.width / 2 - tipWidth / 2;
    left = Math.max(8, Math.min(left, window.innerWidth - tipWidth - 8));
    tip.style.left  = left + 'px';
    tip.style.width = tipWidth + 'px';
    document.body.appendChild(tip);
PLAIN ENGLISH

先拿到术语在屏幕上的位置。

根据窗口宽度算出 tooltip 的宽度。

计算居中的左边距，并限制它不会跑出屏幕。

把 tooltip 固定在 body 上，避免被其他容器遮挡。

🧭
为什么重要？

非技术读者最容易迷路的就是术语。稳定的 tooltip 位置能让他们持续理解。
